# Focus Pet 桌宠动作模组设计文档

本文档重新定义桌宠动作资源包的设计方向：动作模组最多保留 10 个，不再追求覆盖每一种用户状态。状态如何映射到动作，后续再由行为策略决定；当前阶段只设计“值得制作、宠物特色明显、可复用”的核心动作模组。

## 核心原则

1. 最多 10 个动作模组，宁可合并，不做碎片化动作。
2. 优先保留有宠物特征的动作：伸懒腰、跑、爬、扑鼠标、舔毛、睡觉。
3. 删除或合并弱特色动作：进入专注、稳定专注、呼吸、眨眼、醒来、休息结束等不单独作为资源模组。
4. 状态映射暂不细化。比如专注、休息、暂离都可以复用同一个“待机/趴睡/伸懒腰”动作。
5. 所有动作必须适合全局桌面移动，尤其要支持水平移动和垂直移动。
6. 资源包可以继续使用现有 `PetAction` 的 key 做兼容映射，但设计语义以本文档的 10 个动作模组为准。

## 最终动作模组列表

最多 10 个动作模组如下：

| 序号 | 模组名 | 建议资源 key | 主要用途 |
| ---: | --- | --- | --- |
| 1 | 安静待机 | `idle` | 默认状态、低存在感陪伴 |
| 2 | 趴睡/打盹 | `sleep` | 暂离、低能耗、长时间不交互 |
| 3 | 伸懒腰 | `stretch` | 长时间使用后的宠物感提醒 |
| 4 | 水平小跑 | `run` | 桌宠横向移动、跨屏移动 |
| 5 | 垂直攀爬/跳墙 | `climb`，当前可挂到 `screenTransfer` | 桌宠上下移动、贴边上爬、跳上跳下 |
| 6 | 扑向/抱住鼠标 | `cursorPounce`，当前可挂到 `mouseSummon` | 休息回专注亮点、召回、靠近鼠标 |
| 7 | 轻提醒拍爪 | `nudgeGentle` | 温和提醒用户回来 |
| 8 | 强提醒闹腾 | `nudgeStrong` | 明显提醒，但不恐吓 |
| 9 | 放松休息 | `breakRelax` | 休息中、陪用户暂停 |
| 10 | 彩蛋舔毛 | `grooming`，当前可挂到 `blink` 或 `idle` 变体 | 小猫/小狗特色彩蛋动作 |

> 说明：`climb`、`cursorPounce`、`grooming` 是建议新增的语义 key。当前代码未增加这些枚举前，可以先把资源放到兼容 key：`screenTransfer`、`mouseSummon`、`idle` 或 `blink`。

## 被合并或删除的旧动作

| 旧动作 | 处理方式 | 理由 |
| --- | --- | --- |
| `focusStart` | 合并到 `idle` 或 `stretch` | 进入专注不需要单独资源，状态映射后续处理 |
| `focusStable` | 合并到 `idle` | 稳定专注本质是安静陪伴 |
| `breath` | 合并到 `idle` | 呼吸可以作为待机帧的一部分 |
| `blink` | 合并到 `idle` 或 `grooming` | 眨眼太碎，可内嵌到待机循环 |
| `wake` | 合并到 `sleep` 结束段或 `idle` | 不单独做 |
| `breakEnd` | 合并到 `breakRelax` 结束段或 `cursorPounce` | 休息结束的亮点交给扑鼠标 |
| `welcomeBack` | 合并到 `cursorPounce` 或 `idle` | 欢迎回来不是独立动作重点 |
| `distractedLook` | 合并到 `nudgeGentle` | 走神观察和轻提醒可共用 |
| `dragged` | 暂不单独制作 | 拖动时可用 `idle` 静态姿势 |
| `landing` | 暂不单独制作 | 垂直移动落地可放进 `climb` 结尾 |
| `screenTransfer` | 语义替换为 `climb` 或 `run` | 移动方向更重要 |
| `mouseSummon` | 语义替换为 `cursorPounce` | 更有宠物记忆点 |

## 通用美术约束

所有动作模组都继承以下基础提示词：

> 原创 2D 桌宠角色，小猫或小狗气质，圆润小体型，透明背景，正面偏 3/4 视角，适合 macOS 桌面悬浮窗口；动作有动物特色但不吵闹，角色比例稳定，边缘干净，脚底或身体锚点清晰；不要文字、不要 UI、不要复杂背景、不要真实照片、不要版权 IP、不要水印。

通用负面提示词：

> 不要文字、不要 logo、不要复杂背景、不要真实系统鼠标截图、不要强烈闪光、不要高频抖动、不要恐吓表情、不要大幅变形、不要突然换物种、不要遮挡屏幕大面积内容。

## 10 个动作模组详细提示词

### 1. `idle` 安静待机

用途：默认状态、专注时低打扰陪伴、状态未知时 fallback。

动画建议：4-6fps，loop=true，8-16 帧。

提示词：

> 原创 2D 小猫或小狗桌宠安静待机，坐着或趴坐在原地，眼神柔和，身体有非常轻微的呼吸起伏，偶尔尾巴或耳朵轻轻动一下；整体像在安静陪用户工作，不吸引过多注意；透明背景，角色大小一致，锚点稳定，动作幅度很小，不要跳跃，不要文字，不要复杂道具。

### 2. `sleep` 趴睡/打盹

用途：用户暂离、长时间无输入、低能耗状态。

动画建议：2-4fps，loop=true，8-14 帧。

提示词：

> 原创 2D 小猫或小狗桌宠趴着睡觉或蜷成一团打盹，眼睛闭合，呼吸缓慢，耳朵或尾巴偶尔轻动，表情安心；动作非常慢，适合长时间循环，存在感低；透明背景，身体底部锚点稳定，不要翻滚，不要梦境背景，不要文字，不要让角色看起来难过或生病。

### 3. `stretch` 伸懒腰

用途：长时间使用后的宠物特色提醒，也可用于专注/休息之间的过渡。

动画建议：8-10fps，loop=false，14-22 帧。

提示词：

> 原创 2D 小猫或小狗桌宠从坐姿或趴姿慢慢伸懒腰，前爪向前压低，后背拱起，尾巴翘起或轻轻摇动，眼睛眯起，表情舒服；动作有明显动物感，像猫狗刚睡醒或提醒用户活动一下；透明背景，脚底锚点基本稳定，最后回到自然姿势，不要夸张跳跃，不要文字，不要复杂背景。

### 4. `run` 水平小跑

用途：桌宠在屏幕上横向移动、靠近边缘、跨屏水平移动。

动画建议：8-12fps，loop=true，8-16 帧。

提示词：

> 原创 2D 小猫或小狗桌宠做水平小跑动作，短腿快速交替，身体略微前倾，尾巴保持平衡，表情机灵开心；动作循环流畅，适合桌宠从左到右或从右到左移动时播放；透明背景，角色中心和脚底基线稳定，不要真实地面，不要复杂速度线，不要大幅变形，不要文字。

### 5. `climb` 垂直攀爬/跳墙

用途：桌宠需要在屏幕上全局移动时的垂直移动。水平移动用 `run`，垂直移动用 `climb`。包括往上爬、往下跳、贴边登墙、跳上窗口边缘等。

动画建议：10-12fps，loop=false 或短 loop，16-28 帧。建议可拆成上爬版和下跳版，但仍算同一个动作模组。

提示词：

> 原创 2D 小猫或小狗桌宠正在向上攀爬屏幕边缘，前爪扒住一条看不见的竖直边缘，后腿用力蹬，身体一节一节往上挪，尾巴帮助保持平衡；动作像小猫爬窗帘或小狗努力扒墙，非常有动物特色；透明背景，可以暗示一条极简浅色边缘线但不要真实墙面，不要复杂背景；角色不要大幅旋转，锚点随攀爬方向稳定移动，不要文字，不要夸张摔落。

下跳/落下变体提示词：

> 原创 2D 小猫或小狗桌宠从较高位置轻轻往下跳，先探头看一眼，然后后腿发力小跳，落地时身体柔软缓冲，尾巴保持平衡，最后站稳；透明背景，动作短促可爱，不要危险感，不要砸地灰尘，不要文字。

### 6. `cursorPounce` 扑向/抱住鼠标

用途：亮点动作。用户休息后回到专注、召回桌宠、鼠标靠近桌宠时，桌宠扑向鼠标、抱住鼠标或蹭鼠标。

动画建议：10-12fps，loop=false，18-28 帧。

提示词：

> 原创 2D 小猫或小狗桌宠看到鼠标后眼睛一亮，开心地朝鼠标方向小跑两三步，然后轻轻扑到鼠标旁边，用前爪抱住鼠标或贴着鼠标蹭一下，随后松开并坐好；动作亲近、有记忆点，像宠物在和主人的手互动；透明背景，角色不能完全遮住鼠标点击热点，最终回到自然姿势，不要真实系统鼠标截图，可以用极简浅色鼠标指针占位或留空目标方向，不要文字，不要强光效。

分镜：

- 帧 1-4：从待机或休息姿势抬头，发现鼠标。
- 帧 5-10：短距离小跑或小扑。
- 帧 11-18：抱住鼠标、蹭鼠标或两只前爪搭住鼠标旁边。
- 帧 19-24：松开，坐好，尾巴轻轻摆动。

### 7. `nudgeGentle` 轻提醒拍爪

用途：温和提醒。后续可映射到轻度走神、长时间无互动、需要回到任务等场景。

动画建议：8-10fps，loop=false，12-18 帧。

提示词：

> 原创 2D 小猫或小狗桌宠轻轻抬起一只前爪拍一拍空气，或用爪子轻轻点屏幕方向，表情关心但不责备，像是在小声提醒用户；动作温柔、幅度中等、没有警报感；透明背景，角色位置稳定，不要红色警告图标，不要文字，不要生气表情。

### 8. `nudgeStrong` 强提醒闹腾

用途：明显提醒。后续可映射到持续走神、频繁切换、娱乐过久等场景。

动画建议：8-12fps，loop=false，16-24 帧。

提示词：

> 原创 2D 小猫或小狗桌宠更明显地提醒用户，动作可以是小跳一下、两只前爪快速挥动、叼着一个小玩具晃一下，表情认真又可爱；要有存在感，但不能恐吓用户；透明背景，动作清楚，尾巴和耳朵有配合，不要强烈闪光，不要红色警报，不要文字，不要大幅遮挡屏幕。

### 9. `breakRelax` 放松休息

用途：用户休息时、暂停时、低压力陪伴。

动画建议：4-8fps，loop=true，12-24 帧。

提示词：

> 原创 2D 小猫或小狗桌宠进入放松休息状态，可以趴在原地、抱着小杯子、轻轻摇尾巴或把头放在爪子上，表情满足；氛围温暖、恢复精力、无压力；透明背景，动作慢，角色不要离开锚点，不要睡得太死，不要复杂道具，不要文字。

### 10. `grooming` 彩蛋舔毛

用途：宠物特色彩蛋。可在待机很久、用户没有操作、休息中、低频随机触发。这个动作不是功能提醒，而是让桌宠更像真实小动物。

动画建议：6-10fps，loop=false 或 loop=true，16-30 帧。可做多个变体，但仍算一个动作模组。

提示词：

> 原创 2D 小猫或小狗桌宠做动物式舔毛彩蛋动作，小猫可以低头舔前爪、用爪子洗脸，或坐着把后腿张开一点点舔毛；小狗可以舔爪子、挠耳朵、甩甩头；动作真实可爱但要克制，不能低俗，不能过度暴露，整体像自然宠物清洁行为；透明背景，角色比例稳定，动作有生活感，不要文字，不要复杂背景，不要夸张表情。

小猫张腿舔毛变体提示词：

> 原创 2D 小猫桌宠坐在原地，后腿自然微微分开，低头认真舔毛或整理肚子附近的毛，动作笨拙可爱，尾巴轻轻摆动；画面保持健康、自然、宠物日常清洁感，不要拟人化性感姿势，不要低俗，不要夸张暴露；透明背景，角色小体型圆润，动作循环平滑。

  "style": "original_2d_pet",
  "license": "original",
  "distribution": "redistributable",
  "defaultSize": { "width": 160, "height": 160 },
  "anchor": { "x": 0.5, "y": 1.0 },
  "animations": {
    "idle": { "folder": "idle", "fps": 4, "loop": true, "frameCount": 12 },
    "sleep": { "folder": "sleep", "fps": 4, "loop": true, "frameCount": 12 },
    "stretch": { "folder": "stretch", "fps": 10, "loop": false, "frameCount": 18 },
    "run": { "folder": "run", "fps": 10, "loop": true, "frameCount": 12 },
    "screenTransfer": { "folder": "climb", "fps": 12, "loop": false, "frameCount": 24 },
    "mouseSummon": { "folder": "cursorPounce", "fps": 12, "loop": false, "frameCount": 24 },
    "nudgeGentle": { "folder": "nudgeGentle", "fps": 10, "loop": false, "frameCount": 16 },
    "nudgeStrong": { "folder": "nudgeStrong", "fps": 10, "loop": false, "frameCount": 20 },
    "breakRelax": { "folder": "breakRelax", "fps": 6, "loop": true, "frameCount": 16 },

- `screenTransfer` -> `climb`
- `mouseSummon` -> `cursorPounce`
- `blink` 的彩蛋占位 -> `grooming`

## 未来状态映射原则

状态映射之后再细化，但建议遵守以下原则：

- 专注状态不需要 `focusStart` / `focusStable` 两套动作，默认用 `idle`，长时间后偶尔 `stretch`。
- 休息状态用 `breakRelax`，休息结束回专注时可用 `cursorPounce` 作为亮点。
- 暂离状态用 `sleep`。
- 走神提醒只保留 `nudgeGentle` 和 `nudgeStrong` 两级。
- 水平移动用 `run`。
- 垂直移动用 `climb`，包括往上爬、往下跳、贴边登墙。
- 长时间无事件时低频触发 `grooming` 彩蛋。

## 验收清单

- 动作模组总数不超过 10 个。
- 每个动作都具有明显宠物特征，而不是纯 UI 状态标签。
- `climb` 能表达垂直移动，不再用水平跑步硬套上下移动。
- `grooming` 至少包含一个自然舔毛/洗脸/挠耳朵彩蛋变体。
- `cursorPounce` 能表达扑向或抱住鼠标，但不能遮挡点击热点。
- 循环动作首尾帧自然衔接。
- 所有动作透明背景、比例稳定、锚点明确。
- fps 不超过 12，长循环动作建议 4-8fps。
